How to tackle this — recovery strategies by pattern
p. only 1,175 rows — 54.9%

These entries have only a protein description (e.g. p.Arg56fsX157) in the sysname field — no DNA or RNA coordinates. The BED coordinate already places them on the genome; the name just cannot be parsed into REF/ALT.

Fix: write these as coordinate-only BED entries (no REF/ALT), then use the Mutalyzer /position_convert endpoint with the genomic position and the gene's NM_ transcript to obtain a c. notation. The p. in the name can be used as a cross-check label.

r. only 674 rows — 31.5%

RNA-level notations (e.g. r.584_587delagug) describe changes at the transcript but carry no genomic position. The BED coordinate is already the genomic anchor.

Fix: strip the RNA description from the name, treat the BED coordinate as a genomic deletion/insertion of the given length, emit a VCF row with REF=N and ALT derived from the operation type (del → ., ins → sequence). Then let Mutalyzer normalise.

g.IVS / splicing 141 rows — 6.6%

Legacy intronic notation (e.g. g.IVS3-2A>G) that predates HGVS c. intronic offsets. The BED liftover has already placed these on hg38 using the BLAST hit.

Fix: The BED position is reliable — emit it as a plain SNV or indel VCF row anchored to the genomic coordinate, and let Mutalyzer's /normalize convert it to a proper c.-2A>G intronic c. notation.

Compound p. 93 rows — 4.3%

Multi-substitution protein changes (e.g. p.Ala3Val_Thr9Ile_Arg278His) represent haplotypes — multiple variants on one allele. Each sub-change maps to a different position.

Fix: split on _, resolve each amino-acid substitution individually against the transcript using Mutalyzer /back_translate, and emit one VCF row per sub-variant.

No sysname 33 rows — 1.5%

Entries where the pub.html had no Systematic name at all — the BED name is just an accession like L0001. Without any mutation description, the REF/ALT cannot be inferred.

Fix: look up each accession in the original pub.html source file and retrieve the mutation description manually, or flag these for MV review. Only 33 rows — manual curation is feasible.
